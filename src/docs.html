Installation

To add Myworkout Styles Library to a project use Node.js:

npm install git+ssh://git@bitbucket.org/myworkout/myworkout-styles-library.git --save

Library delivers SASS, so it expects to be compiled by client application. However, it also provides common assets (images) and it should be available in production too (hence --save).

Setup

Top level SASS file of client application should import Myworkout Styles Library like this: 

@import "path/to/node_modules/normalize.css/normalize.css";
@import "path/to/node_modules/myworkout-styles-library/src/all";

It is recommended to use normalize.css together with this library. In a case of using 3rd party frameworks like Zurb Foundation or Bootstrap, this is not needed since they already import it.

Old browsers

Some features are not supported by some older browsers. To use fallback CSS add .mw-fallback-css on body for old browsers:

  • IE9 and lower
  • Chrome v25.0.0 and lower
  • Firefox v15.0.0 and lower
  • Opera v29.0.0 and lower

Base rules

Base rules are the defaults. They are almost exclusively single element selectors but it could include attribute selectors, pseudo-class selectors, child selectors or sibling selectors.

Base rules for Myworkout profile's global scope are defined in src/sass/base.scss file.

Typography

Typography rules specified in this library are related to the scope of an article or .mw-article. However it is possible to use these styles outside of article context, by using provided classes.

.mw-text - Default text style
.mw-title - Default style for a page title. Inside article or .mw-article this is default style for h1.
.mw-subtitle - Default style for subtitles. Inside article or .mw-article this is default style for h2.
.mw-heading - Default style for third level headings. Inside article or .mw-article this is default style for h3.
.mw-lead - Default style for intro paragraph.
.mw-p - Default style for a paragraph. Inside article or .mw-article this is default style for p.
.mw-quote - Default style for quotes. Inside article or .mw-article this is default style for blockquote.

Form elements: Text inputs

Text inputs are supposed to be used mainly with inputs of type text, password, email and similar. It is recommended to use them with .mw-full-width class for device responsivity.

label.mw-label
input.mw-input
div.mw-special-input-wrap input *2
input.mw-input.mw-error *3
div.mw-special-input-wrap.mw-error input *2 *3
input.mw-input.mw-full-width *4
div.mw-special-input-wrap.mw-full-width input *2 *4
input.mw-input.mw-full-width.mw-error *3 *4
div.mw-special-input-wrap.mw-full-width.mw-error input *2 *3 *4
*2 This element requires an HTML container as wrapper. 
<div class="mw-input-wrap">
    <input>
</div>
If input should be disabled, in addition to disabled attribute on input there should be mw-disabled class on the wrapper.
*3 Meant to be used for raising a warning or indicating an error by using JavaScript to add mw-error class on input (basic inputs) or wrapper (special ones).
*4 This input takes 100% width of its positioned parent, by adding mw-full-width class on input (basic inputs) or wrapper (special ones).

  • Do nothing. IE 10/11 will render background in one appropriate color.
